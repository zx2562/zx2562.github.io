Arcdps - 团队DPS监控插件

  2019.10.18

简介

  • 安装任何插件前，注意一定要绑定 口袋密令，谨防盗号。
  • Arcdps 是一款团队DPS监控插件，其工作原理是通过获取本地的战斗记录信息得到伤害数据，该数据可能与真实数据略有偏差，仅供参考使用。
  • 2017年2月，ArenaNet 已经允许第三方插件可以通过战斗记录信息获取数据，它不会改变客户端的文件，正常使用不会有封号风险。

注意！

插件安装前，确保游戏安装路径不存在中文，否则安装插件后启动游戏会导致插件乱码。
如果有安装 Reshade 滤镜 插件，请先将滤镜插件在游戏根目录下 bin64 文件夹内的 d3d9.dll 和 d3d9.ini 文件名称分别改为 Reshade64.dll 和 Reshade64.ini。
在游戏版本更新后，该插件必须更新覆盖到对应版本，否则会导致游戏报错。

  • 第一步，退出正在登录中的游戏客户端，并关闭游戏登录界面。
  • 第二步，解压 Arcdps插件压缩包，解压后如图所示。

  • 第三步，将解压后的 addons文件夹（汉化包文件） 放在游戏根目录下，游戏根目录及路径不能存在中文。

  • 第四步，将解压后的 bin64文件夹 与游戏跟目录下的 bin64 文件夹覆盖。

  • 游戏启动后，会在 文档 中生成以下文件。至此 Arcdps 已经安装完毕。

  • 初次使用时，插件菜单可能出现必须用 Shift + Alt + 鼠标左键 才能拖动的情况，此时按 Shift + Alt + T 打开 设置菜单，点灭 鼠标穿透 的启用，即可用鼠标左键直接拖动菜单。

  • 将最新版本压缩包内的 bin64 文件夹与游戏根目录下的 bin64 合并替换即可。

滚动战斗文本

  • 下载（百度网盘）：点击查看，提取码：4a9l
  • 用于滚动显示战斗文本，高度可自定义。左侧为受到的伤害 及 治疗，右侧为输出的伤害。
  • 如果要添加自定义字体，请将其复制到游戏根目录下的 addons / sct / fonts 子文件夹中。
  • 您可以通过快捷键 Alt + Shift + T 打开 SCT Options，修改默认参数。

大型地下城机制记录器（英文）

  • 下载（百度网盘）：点击查看，提取码：4a9l
  • 本插件用于记录大型地下城中玩家与 BOSS 机制互动的记录信息，例如：蓝圈被传送、炸弹被谁接到、毒圈是谁排放的。
  • 使用热键打开 Alt + Shift + L（日志）/ Alt + Shift + N（图表）

快速卸载（有残留文件）：

  • 如果想要快速卸载，请直接删除游戏根目录下的 bin64 的文件夹。此文件夹中的原文件，会在游戏启动后重新生成。

完全卸载：

  • 如果你希望完全卸载此插件，请先将游戏根目录下的 bin64 文件夹删除，再将以下几张图片中蓝色背景文件删除即可。

  • Shift + Alt + H ：插件全局 显示 / 隐藏
  • Shift + Alt + T ：设置菜单（OPTIONS） 显示 / 隐藏
  • Shift + Alt + B ：BUFF面板（BUFF TABLE） 显示 / 隐藏
  • Shift + Alt + C ：团队统计（AREA STATS） 显示 / 隐藏
  • Shift + Alt + S ：个人统计（SELF） 显示 / 隐藏
  • Shift + Alt + K ：统计信息 锁定 / 解锁
  • Shift + Alt + O ：重新加载（某些）配置
  • Shift + Alt + M ：坐骑按键设置
  • ESC ：隐藏非永久菜单

注意：插件中每个表右上方的 × 按下一次，都等同于使用快捷键隐藏掉了该表，例如按 × 隐藏了团队统计，则可以使用 Shift+Alt+C 重新显示该表。而一张表要正常显示，不仅需要表状态是显示状态，而且需要插件全局（Shift+Alt+H）也是显示状态。

生成日志设置

  • 按 Shift + Alt + T 打开 设置菜单，找到 日志选项，点亮下图红框中的选项，即可在战斗结束后自动保存战斗日志。

  • 战斗日志会保存在 文档 的下述位置，WIN10系统 可以保存压缩文件进去，WIN7系统 则保存未压缩的原文件，请定时清理该文件夹，否则体积会变得超大。

日志生成触发条件

在最新的 Arcdps版本中，已经支持对 WvW 模式的日志记录

  • PVE模式：在附近拥有已被 Arcdps识别的 BOSS ID时，Arcdps 会在 团队进入战斗 直至 团队所有玩家脱离战斗 为止，生成战斗日志。
  • WvW模式：由于Arcdps拥有一个记录半径，你需要在大团中心的位置才能较准确的记录战斗日志。在大团中心附近的玩家 进入战斗开始 至 大团中心附近的玩家脱离战斗 为止，生成战斗日志。
  • 自动压缩需要系统具有 compress-archive 的 Powershell，因此目前只有 Windows10 操作系统支持。

方法一，上传服务器解析：

  • 将 arcdps 生成的 .evtc，.evtc.zip 或 .zevtc 文件上传至：dps.report（英文）
  • 待服务器将日志文件解析完毕后，可以将解析后的战斗记录网址发送给其他团队成员查看。

方法二，战斗日志解析（神油汉化）：

  • 第一步，下载 战斗日志解析器，下载地址：百度网盘，提取码：l73b
  • 第二步，根据最近的战斗时间，从 我的文档 下述位置选择并解压战斗日志文件。

文档 / Guild Wars 2 / addons / arcdps / arcdps.cbtlogs

  • 第三步，将解压出的 .evtc 文件拖动到 GuildWars2EliteInsights.exe 的图标上读取解析，解析后会生成一个 .html 文件。
  • 第四步，将生成的 .html 文件使用浏览器打开即可查看日志。

其他方法：

  • GW2 Raidar Uploader：www.gw2raidar.com（官网 英文），Github页面
  • GW2 Elite Insights Parser：Github页面
  • L0G-101086：Github页面
  • Arcdps 通过两个数据源获得战斗信息：区域性战斗信息 = 服务器信息反馈 + Buff模拟，个人战斗信息 = 聊天记录中的战斗信息。
  • 区域型技能的统计在激活时可能会丢失一些因触发而产生的 困惑 ，但不会丢失那些快速造成多段打击的伤害统计。
  • 如果10名玩家同时使用技能，还要考虑在目标死亡的瞬间是否有伤害统计溢出 或 治疗溢出。
  • 如果10名玩家同时自动攻击目标，直到目标死亡，10名玩家只有一人可以在击杀的瞬间记录伤害。如果10名玩家都对目标造成症状伤害，则10名玩家都会在击杀瞬间记录伤害。
  • 这两种伤害计算方式与实战伤害相接近，但想要100%记录伤害及治疗信息是不能做到的。
